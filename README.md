## Бізнес-проблема
**Утримання клієнтів (Retention)** коштує в 5-10 разів дешевше, ніж залучення нових. У цьому проєкті я досліджую дані про поведінку користувачів сервісу, щоб виявити "червоні прапорці" — ознаки того, що клієнт збирається піти, та надати бізнесу конкретні рекомендації щодо їх утримання.
### 🛠 Технічний стек
* **Інструменти:** Python (Pandas для обробки, Seaborn/Matplotlib для візуалізації, Scikit-learn — логістична регресія).
* **Методологія:** RFM-аналіз (Recency, Frequency, Monetary), кореляційний аналіз, когортний аналіз.
* **Дані:** Датасет з інформацією про транзакції, тип контракту та активність клієнтів.
### 💡 Етапи роботи
1. **Data Audit та очищення**
* **видалення null CustomerID**
* **фільтрація повернень і нульових цін**
* **створення Revenue**
2. **Customer-level агрегація**
* **Recency, Frequency, Monetary**
* **Lifetime, Avg order value**
3. **Churn definition**
* **churn = відсутність покупок > 180 днів**
4. **RFM-сегментація**
* **виділення Champions, At Risk, Lost тощо**
5. **Retention / Cohort analysis**
* **аналіз повернення клієнтів у часі**
6. **Correlation & Predictive modeling**
* **визначення драйверів churn**
* **побудова логістичної моделі**
### Виклики
* **Велика кількість транзакцій без CustomerID (~25%)**
* **Наявність повернень і від’ємних значень**
* **Багато одноразових покупців (складність прогнозу)**
* **Дисбаланс класів churn / active**
### 📈 Головні інсайти та рекомендації
* **Інсайт 1:** Клієнти, які користуються сервісом менше ніж 4 місяці та мають місячний контракт, становлять 65% всього відтоку.
* **Інсайт 2:** Наявність підключеної послуги "Технічна підтримка" знижує вірогідність відтоку на 30%.
### Рекомендація для бізнесу:
1. Впровадити автоматичну знижку або бонусний період при переході з місячного на річний контракт.
2. Налаштувати тригерну розсилку для сегмента "Під загрозою" з пропозицією безкоштовної консультації техпідтримки.

## Дані
Для аналізу використано відкритий набір даних Online Retail Dataset з репозиторію UCI Machine Learning. Чому обрано саме цей датасет:
* Транзакційна структура: Містить понад 500,000 записів про реальні операції британського онлайн-ритейлера за 2010–2011 роки.
* Придатність для RFM: Наявність ідентифікаторів клієнтів (**CustomerID**), дат (**InvoiceDate**) та сум (**Quantity × UnitPrice**) дозволяє розрахувати ключові метрики лояльності: **Recency** (давність), **Frequency** (частота) та **Monetary** (грошова вартість).
* Географічне різноманіття: Дані охоплюють понад 30 країн, що дає можливість провести кореляційний аналіз між регіоном та обсягом продажів.
